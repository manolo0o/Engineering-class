Infografía — Cómo publicar tu HTML en GitHub

Guía rápida en 6 pasos con puntos clave y recursos — lista para probar.

Paso 1 — Prepara tus archivos

Coloca index.html en la raíz. Si tienes CSS o JS, ponlos en carpetas css/ y js/. Usa rutas relativas para que todo funcione tras subirlo.

Paso 2 — Crea el repositorio

En GitHub: New → repository. Nombra el repo (por ejemplo mi-sitio) y súbelo. O usa git init local y luego git push al remoto.

Paso 3 — Sube los archivos

Usa "Upload files" en la web o desde la terminal: git add . → git commit -m "Initial" → git push.

Paso 4 — Activa GitHub Pages

En Settings → Pages selecciona la rama main (o gh-pages) y la raíz / (root). Espera unos minutos y tu sitio estará disponible.

Paso 5 — Verifica rutas y recursos

Comprueba que las imágenes, tipografías y archivos estén con rutas relativas (./img/logo.png) y que no dependan de rutas del sistema local.

Paso 6 — Comparte la URL

Tu URL será https://TU-USUARIO.github.io/NOMBRE-DEL-REPO/. Si el repo se llama TU-USUARIO.github.io, será la raíz del dominio.

Verificación

¿No aparece el sitio?

Revisa que index.html esté en la raíz, y el branch correcto en Pages. A veces tarda unos minutos en desplegar.

Imágenes no cargan

¿Tienes rutas absolutas? Cámbialas por relativas o súbelas al repo.

SPA (React, Vite)

Requiere build y a veces 404.html o configuración de base path.

Consejos rápidos
  • Usa index.html como punto de entrada.
  • Evita rutas absolutas al disco.
  • Si usas frameworks, genera la carpeta build/ o dist/.
